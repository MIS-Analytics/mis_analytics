DeSAPher

SAP Data Dictionary Explorer. Decipher cryptic SAP column names to something more descriptive.

The Problem

Working with SAP systems often means dealing with hundreds of tables with cryptic names like MARA, VBAK, or MAKT. Understanding what these tables contain and how they relate to each other traditionally requires extensive documentation lookup or SAP expertise.

Our Solution

We created a tool that automatically fetches and structures SAP table definitions from online documentation. This gives us instant access to: - Field names and their meanings - Data types and lengths - Relationships between tables (via check tables) - Comprehensive descriptions of each field - Table names and descriptions

Benefits

  • Time Saving: No more manual documentation lookups
  • Better Understanding: Clear visibility of table structures and meanings
  • Easier Data Analysis: Quick reference for field names and their purposes
  • Knowledge Sharing: Makes SAP data structures more accessible to team members

Future Directions

1. Interactive Interface

  • Build a searchable interface for quick table/field lookups
  • Implement full-text search across descriptions
  • Add semantic search using LLMs to find relevant tables by describing needs in plain English

2. Visual Data Model

  • Create interactive graph visualizations showing table relationships
  • Highlight primary/foreign key connections
  • Enable visual exploration of the SAP data model

3. AI-Powered Data Assistant

  • Use LLMs with our structured documentation as context
  • Generate SQL queries from natural language questions
  • Suggest relevant tables for specific business questions
  • Provide data model explanations in plain language

Our plan

As data scientists working with ERP systems like SAP, we often encounter large datasets with hundreds of cryptically named columns. To better understand these data sources, we aim to web scrape SAP’s data dictionary documentation to create programmatic access to table definitions, column descriptions, and data types.

Steps

  1. Scrape column description for one table (for example, MARA)
  2. Bring it in to a easily serachable format
  3. Apply process to all tables

from pathlib import Path
CACHE_PATH = Path('sap_sheet.parquet')
def get_sap_tables_structure(tables, cache_path=CACHE_PATH, force_refresh=False):
    """
    Gets structure for multiple SAP tables, caching results in a single parquet file.
    Only scrapes tables not already in the cache. force_refresh=True re-scrapes everything.
    """
    if force_refresh and cache_path.exists():
        cache_path.unlink()

    # Load existing cache
    if cache_path.exists():
        cached = pd.read_parquet(cache_path)
        already_cached = set(cached['Table'].unique())
    else:
        cached = None
        already_cached = set()

    # Scrape only missing tables
    missing = [t for t in tables if t.upper() not in already_cached]
    new_dfs = []
    for table in missing:
        url = get_sap_table_url(table)
        df = get_sap_table_structure(url)
        if df is not None:
            df['Table'] = table.upper()
            df['Table Description'] = get_sap_table_description(url)
            new_dfs.append(df)

    # Combine and save
    all_dfs = [d for d in [cached] + new_dfs if d is not None]
    if not all_dfs:
        return None
    result = pd.concat(all_dfs, ignore_index=True)
    if missing:  # only write if something new was added
        result.to_parquet(cache_path, index=False)

    # Return only requested tables
    return result[result['Table'].isin([t.upper() for t in tables])]

A few good options, from simplest to most robust:

  1. Save to Parquet/CSV — scrape once, save to file, load from file on subsequent runs. Parquet is nice because it preserves dtypes.
  2. Pickle the DataFrame — similar but keeps all Python types exactly.
  3. SQLite — good if you want to query the data without loading it all into memory.

For your use case, I’d recommend Parquet — it’s compact, fast, preserves types well, and you can easily inspect it with pandas. The pattern would be: check if the file exists → if yes, load it; if not, scrape and save.

Something like:

from pathlib import Path

CACHE_PATH = Path('sap_sheet.parquet')

def get_or_scrape_sap_tables(tables, cache_path=CACHE_PATH):
    if cache_path.exists():
        return pd.read_parquet(cache_path)
    df = get_sap_tables_structure(tables)
    df.to_parquet(cache_path, index=False)
    return df

CACHE_PATH = Path('sap_sheet.parquet')

def get_sap_tables_structure(tables, cache_path=CACHE_PATH, force_refresh=False):
    """
    Gets structure for multiple SAP tables, caching results in a single parquet file.
    Only scrapes tables not already in the cache. force_refresh=True re-scrapes everything.
    """
    if force_refresh and cache_path.exists():
        cache_path.unlink()

    # Load existing cache
    if cache_path.exists():
        cached = pd.read_parquet(cache_path)
        already_cached = set(cached['Table'].unique())
    else:
        cached = None
        already_cached = set()

    # Scrape only missing tables
    missing = [t for t in tables if t.upper() not in already_cached]
    new_dfs = []
    for table in missing:
        url = get_sap_table_url(table)
        df = get_sap_table_structure(url)
        if df is not None:
            df['Table'] = table.upper()
            df['Table Description'] = get_sap_table_description(url)
            new_dfs.append(df)

    # Combine and save
    all_dfs = [d for d in [cached] + new_dfs if d is not None]
    if not all_dfs:
        return None
    result = pd.concat(all_dfs, ignore_index=True)
    if missing:  # only write if something new was added
        result.to_parquet(cache_path, index=False)

    # Return only requested tables
    return result[result['Table'].isin([t.upper() for t in tables])]

The key things it does: - Loads the cache and checks what’s already there - Only scrapes the missing tables - Appends new tables and saves back - Returns only the tables you asked for (even if the cache has more) - force_refresh=True deletes the cache and starts fresh
